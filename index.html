01 / Discovery

Adaptive source discovery

  • Unlimited target sites with active-target selection and latest-check status.
  • robots.txt hints, homepage sitemap declarations, sitemap indexes, WordPress paths, Google News paths, and common sitemap URLs.
  • Versioned multilingual publisher catalog with language-specific sitemap priority and recent-first dated sitemap traversal.
  • Regular XML and gzip sitemap parsing with normalized candidate URLs.
  • Deterministic news URL scoring with optional AI URL classification.
  • Seen-URL deduplication and unseen-candidate backfill when fresh selections run low.
02 / Fetch and network

HTTP and crawl IP operations

  • Configurable concurrency, timeout, user agent, article limit, sitemap limit, and score threshold.
  • Exact HTTP codes, status families, redirects, final URLs, content type, and request duration.
  • Unlimited credential-aware HTTP(S) proxies with optional direct server-IP fallback.
  • Request rotation and retries for transport errors, timeouts, 403, 407, and 429 responses.
  • Persistent health, latency, egress IP, quarantine, cooldown, blacklist, reset, and capacity reporting.
  • Per-target Basic, Bearer, custom-header, and session-cookie access with explicit host allowlists and cross-host redirect stripping.
03 / Extraction and dataset

Publisher-aware article records

  • Title selection from Open Graph, Twitter, JSON-LD, HTML title, H1, and decoded URL slugs.
  • Canonical URL, description, language, text preview, content length, content hash, and timing fields.
  • Publisher categories and tags from article metadata, keywords, news keywords, and JSON-LD.
  • Historical title and taxonomy backfill for previously collected records.
  • Local append-only JSONL, on-demand CSV export, raw HTML retention, and optional ingestion submission.
04 / AI provider layer

Multi-provider AI routing

  • Unlimited providers with independent API keys, base URLs, models, and enabled states.
  • Separate routing for URL classification, sitemap analysis, article extraction, and summarization.
  • Responses-compatible structured extraction for summaries, entities, and topical tags.
  • Provider health, last HTTP status, last check, last error, missing-key, quota, and wallet warnings.
  • Graceful deterministic crawling and base record storage when optional AI is unavailable.
05 / Developer platform

Authenticated news APIs

  • Developer signup, login, account sessions, plan selection, and self-service dashboard.
  • Read-only product preview accounts with email/password access and backend-enforced write restrictions.
  • One-time key display, hashed secret storage, scopes, revocation state, and last-use tracking.
  • X-API-Key and Bearer authentication for paginated list, article detail, usage, and crawl-plan APIs.
  • Dataset search plus source, publisher category, and publisher tag filters without raw HTML exposure.

Frequently asked questions

Technical and commercial questions, answered clearly.

Concise answers for teams evaluating the crawler, Developer API, deployment model, and product experience.

What is Adaptive News AI?

Adaptive News AI is news intelligence infrastructure that discovers sources, crawls articles, enriches records, schedules recurring jobs, and serves controlled datasets through a Developer API.

Can the crawler work without an AI provider?

Yes. Deterministic sitemap discovery and URL scoring continue without AI. Optional providers improve URL classification, extraction, summarization, entity detection, and tagging.

How does crawl IP management work?

The operator can configure multiple HTTP or HTTPS proxies, retain direct server fallback, rotate requests, quarantine unhealthy endpoints, detect blacklist conditions, and review capacity recommendations.

What does the Developer API expose?

Scoped API keys provide controlled article records, search and source taxonomy filters, usage reporting, and crawl-plan operations. Raw HTML and operator state are excluded.

Can Adaptive News AI run on one customer server?

Yes. The platform is designed for a lean Docker-based single-server deployment with persistent local state and optional integration with a separate ingestion service.

Can I preview the dashboard before deployment?

Yes. A dedicated email/password preview shows realistic sources, crawl plans, AI health, usage, and dataset records. Preview accounts are read-only and cannot create keys, run crawls, or change production settings.

Can it crawl sources that require an authorized account?

Yes. Operators can attach host-scoped Basic, Bearer, custom-header, or existing session-cookie profiles to targets they are authorized to access. The crawler does not bypass paywalls or publisher controls.
